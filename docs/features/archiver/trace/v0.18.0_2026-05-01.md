# v0.18.0 存档快照 — 本地缓存与离线同步

日期：2026-05-01
节点总数：116（新增 4，扩展 3）

## 变更摘要

引入本地数据库（drift + SQLite），实现本地优先的数据读写。打开 app 瞬间展示会话列表和聊天记录，离线时可浏览已缓存数据，重连后自动增量同步。

后端改动极小：GET /messages 新增 after_seq 参数。前端新建 flash_im_cache 模块（LocalStore 抽象接口 + DriftLocalStore 实现 + SyncEngine 同步引擎），改造三个 Repository 的读取链路。

## 关键设计决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 存储层抽象 | LocalStore 接口 + DriftLocalStore 实现 | 上层不绑定 ORM，未来可替换 |
| 变更通知 | SyncEngine 回调（腾讯 IM SDK 模式） | 同步完成后直接通知 Cubit |
| Repository 改造 | 本地优先 + 空数据 fallback HTTP | 有数据秒开，首次不白屏 |
| 写入策略 | upsert（insertOnConflictUpdate） | 幂等，WS 推送和增量同步不重复 |
| 数据库隔离 | per-user 文件（im_cache_{userId}.db） | 切换用户无需清理 |
